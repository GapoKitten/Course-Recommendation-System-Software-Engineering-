# 🎓 Course Recommendation System - Software Engineering Project

This repository contains the complete **Software Engineering** documentation for a "Course Recommendation System". This system was designed to assist Computer Science students in planning their course registrations effectively.

This project demonstrates a full Software Development Life Cycle (SDLC) understanding, from requirement gathering to software testing.

## 📌 Project Overview
The objective of this project is to solve the problem of students taking the wrong prerequisite courses, dropping classes, or delaying graduation. The proposed system recommends appropriate courses, visualizes study plans, and saves time for both students and the department.

## 🛠️ Software Engineering Skills Demonstrated
This repository showcases the following core software engineering competencies, divided into 7 key phases:

* **[Task 1] Project Planning & Business Value:** Defining stakeholders, business objectives, and success measurement metrics.
* **[Task 2] Requirement Analysis:** Gathering and documenting detailed Functional and Non-Functional Requirements (e.g., Usability, Performance, Security).
* **[Task 3] Use Case Modeling:** Creating Use Case Diagrams and writing detailed Use Case Descriptions (e.g., Login, Recommend Course, Manage Subject).
* **[Task 4] Object-Oriented Design:** Designing the system architecture using a **Class Diagram** (Staff, Student, Course, Passed Course).
* **[Task 5] Dynamic Modeling:** Mapping system behaviors and interactions using **Sequence Diagrams**.
* **[Task 6] User Interface (UI) Design:** Creating Interface Structure Diagrams and designing screen prototypes for both Student and Admin roles.
* **[Task 7] Software Testing & QA:** Executing test cases, documenting Bug Reports, and verifying resolutions using standard QA practices.
